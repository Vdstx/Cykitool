Nikto

About Nikto

Nikto is an open-source web server scanner that performs comprehensive tests against web servers for multiple items, including over 6,700 potentially dangerous files/programs, checks for outdated versions of over 1,250 servers, and version-specific problems on over 270 servers. Nikto is a widely used tool in penetration testing and security assessments to identify vulnerabilities and misconfigurations in web servers.


Installation

Nikto is written in Perl and requires Perl to be installed on your system.


Installing Nikto on Debian-based Systems:

sudo apt-get install nikto

Installing Nikto from Source:

git clone https://github.com/sullo/nikto.git
cd nikto/program
perl nikto.pl -h

Examples of Nikto Usage

Basic Scan of a Website:

nikto -h http://example.com
  • -h: Specifies the target host to scan.

Specifying a Port:

nikto -h http://example.com -p 8080
  • -p: Specifies the port to scan (default is 80 for HTTP).

Scanning Multiple Hosts from a File:

nikto -h hosts.txt
  • Where hosts.txt is a file containing a list of hosts or IP addresses.

Saving the Scan Results to a File:

nikto -h http://example.com -o scan_results.txt -Format txt
  • -o: Specifies the output file.
  • -Format: Specifies the output format (txt, csv, html, xml, nbe).

Using SSL/TLS Scanning:

nikto -h https://example.com -ssl
  • -ssl: Forces Nikto to use SSL when connecting to the server.

Ignoring a Specific Check:

nikto -h http://example.com -id 001234
  • -id: Specifies an ID of a check to disable.

Useful Options

Option Description
-h, --host Target host(s) to scan (hostname, IP, or file).
-p, --port Port(s) to scan (e.g., 80,443).
-Tuning Scan tuning options to control test types (e.g., 1 for interesting files).
-o, --output Output file to write the results.
-Format Format of the output file (e.g., txt, html, csv).
-ssl Force SSL mode on port.
-nossl Disable SSL.
-timeout Timeout for requests (default 10 seconds).
-useragent Set the User-Agent string.
-useproxy Use the proxy defined in the configuration file.
-list-plugins List all available plugins and their status.
-update Update the plugins and databases.
-evasion Specify evasion technique(s) to use (e.g., 1 for random URI encoding).
-Plugins Select which plugins to run (e.g., all, outdated, headers).

Troubleshooting and Tips

  • Proxy Support: Configure proxy settings in the Nikto configuration file or use the -useproxy option.
  • Increasing Verbosity: Use the -v option to increase verbosity for more detailed output.
  • Custom Headers: Use the -useragent option to set a custom User-Agent string, or modify headers in the configuration file.
  • Evasion Techniques: Employ evasion techniques with the -evasion option to bypass basic IDS/IPS detection.
  • Updating Nikto: Regularly update Nikto's plugins and databases using the -update option to ensure up-to-date vulnerability checks.
  • Scan Tuning: Use the -Tuning option to control the types of tests performed, focusing on specific vulnerability categories.

Legal Considerations

Ensure you have explicit permission to scan and test the target web servers. Unauthorized scanning may violate laws, regulations, and ethical guidelines.
