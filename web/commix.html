Commix

About Commix

Commix (short for [comm]and [i]njection e[x]ploiter) is an open-source penetration testing tool that automates the process of detecting and exploiting command injection vulnerabilities in web applications. It is designed to help security professionals and researchers test web applications for vulnerabilities related to command injection attacks.


Installation

Commix is written in Python and supports both Python 2 and Python 3. You can install it by cloning the repository from GitHub:

git clone https://github.com/commixproject/commix.git
cd commix
sudo python setup.py install

Alternatively, you can run Commix directly without installation:

python commix.py [options]

Examples of Commix Usage

Basic Command Injection Testing:

python commix.py --url="http://example.com/vulnerable.php?id=1"
  • --url: Specifies the target URL to test for command injection.

Providing Data via POST Method:

python commix.py --url="http://example.com/login.php" --data="username=admin&password=admin"
  • --data: Specifies data to be sent via POST request.

Setting a Custom User-Agent:

python commix.py --url="http://example.com/" --user-agent="Mozilla/5.0"

Using Tor Network:

python commix.py --url="http://example.com/" --tor
  • --tor: Routes traffic through the Tor network for anonymity.

Enumerating System Information:

python commix.py --url="http://example.com/vuln.php?id=1" --sys-info
  • --sys-info: Retrieves system information from the target.

Useful Options

Commix offers a wide range of options to customize the testing process. Here are some of the most commonly used options categorized for better understanding:

General Options

  • -h, --help: Show the help message and exit.
  • -v VERBOSE: Set verbosity level (0-4). Default is 0.
  • --version: Show the program's version number and exit.
  • --update: Check for updates and apply if any are available.
  • --batch: Run in non-interactive mode using default options.

Target Options

  • -u URL, --url=URL: Specify the target URL.
  • -r REQUESTFILE: Load HTTP request from a file.
  • -d DATA, --data=DATA: Data string to be sent through POST.
  • --method=METHOD: Force the usage of a specific HTTP method (e.g., GET, POST).

Request Options

  • --user-agent=AGENT: Specify a custom User-Agent header.
  • --random-agent: Use a randomly selected User-Agent header.
  • --cookie=COOKIE: Specify HTTP Cookie header.
  • --proxy=PROXY: Use a proxy to connect to the target URL (e.g., http://127.0.0.1:8080).
  • --tor: Use the Tor network for anonymity.
  • --auth-cred=AUTH_CRED: HTTP authentication credentials (e.g., "admin:admin").

Injection Options

  • -p TEST_PARAMETER: Specify parameter(s) to test for injection.
  • --suffix=SUFFIX: Injection payload suffix string.
  • --prefix=PREFIX: Injection payload prefix string.
  • --technique=TECH: Specify injection techniques to use (e.g., classic, time-based).
  • --os-cmd=OS_CMD: Execute a single OS command on the target.
  • --tamper=TAMPER: Use tamper scripts to bypass filters (e.g., space2comment).

Enumeration Options

  • --current-user: Retrieve the name of the current user.
  • --hostname: Retrieve the hostname of the target system.
  • --is-root: Check if the current user has root privileges.
  • --users: Retrieve a list of users on the target system.
  • --passwords: Retrieve password hashes of system users.

File Access Options

  • --file-read=FILEPATH: Read a file from the target host.
  • --file-write=FILEPATH: Write to a file on the target host.
  • --file-upload=FILENAME: Upload a file to the target host.
  • --file-dest=DEST_PATH: Specify the destination path on the target host for writing or uploading files.

Troubleshooting and Tips

  • Bypassing WAFs: Use tamper scripts with the --tamper option to bypass Web Application Firewalls.
  • Verbose Output: Increase verbosity with -v to get more detailed output for debugging.
  • Session Management: Use the -s option to save sessions and --flush-session to remove them.
  • Custom Headers: Add extra HTTP headers using the -H option.
  • Timeouts and Retries: Adjust --timeout and --retries options for unstable network conditions.

Legal Considerations

Always ensure you have explicit permission to test the security of a web application. Unauthorized testing may violate laws and ethical guidelines.
