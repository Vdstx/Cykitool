Gobuster

About Gobuster

Gobuster is a command-line tool written in Go, used for brute-forcing URLs (directories and files) and DNS subdomains on web servers. It is designed for speed and efficiency, making it a popular choice for penetration testers and security researchers to discover hidden content and subdomains during web application assessments.


Installation

Prerequisites: Ensure you have Go installed on your system (version 1.16 or higher).


Installing Gobuster:

go install github.com/OJ/gobuster/v3@latest

This will install Gobuster in your Go binaries directory, typically $GOPATH/bin.


Alternative Installation using Binary Releases:

  • Download the precompiled binaries from the official GitHub releases page.
  • Extract the archive and move the executable to a directory in your system's PATH.

Examples of Gobuster Usage

Directory Brute-Forcing

Basic Directory Scan:

gobuster dir -u http://example.com -w /path/to/wordlist.txt
  • dir: Specifies the mode for directory brute-forcing.
  • -u: The target URL.
  • -w: Path to the wordlist file.

DNS Subdomain Brute-Forcing

Basic DNS Scan:

gobuster dns -d example.com -w /path/to/wordlist.txt
  • dns: Specifies the mode for DNS subdomain brute-forcing.
  • -d: The target domain.
  • -w: Path to the wordlist file.

Virtual Host Brute-Forcing

VHost Enumeration:

gobuster vhost -u http://example.com -w /path/to/wordlist.txt
  • vhost: Specifies the mode for virtual host brute-forcing.
  • -u: The base URL of the target.
  • -w: Path to the wordlist file.

Useful Options

Option Description
-u, --url The target URL or domain.
-w, --wordlist Path to the wordlist file.
-t, --threads Number of concurrent threads (Default: 10).
-o, --output File to write output results.
-e, --expanded Show full URLs in output.
-k, --no-tls-validation Skip TLS certificate verification.
-x, --extensions File extension(s) to search for (e.g., php,txt).
-s, --status-codes Filter results by HTTP status codes (e.g., 200,204,301).
-b, --hide-length Hide the length of the body in the output.
--timeout Request timeout in seconds (Default: 10).
--proxy Use a proxy for requests (e.g., http://127.0.0.1:8080).
-H, --header Add custom HTTP headers (e.g., "Authorization: Bearer TOKEN").
-a, --user-agent Set the User-Agent string.
--cookies Set cookies for the requests.
--follow-redirect Follow HTTP redirects.

Troubleshooting and Tips

  • Wordlists: Use comprehensive wordlists for better results. Popular choices include those from SecLists.
  • File Extensions: Specify common file extensions using the -x option to discover hidden files (e.g., -x php,html,txt).
  • Rate Limiting: Adjust the number of threads with -t to respect the target's rate limits and avoid being blocked.
  • Proxy Usage: Use the --proxy option to route traffic through tools like Burp Suite for deeper analysis.
  • Status Codes: Use the -s option to filter results based on HTTP status codes of interest.
  • HTTPS Issues: If you encounter TLS errors, use -k to skip TLS verification.

Legal Considerations

Ensure you have explicit permission to perform scanning and enumeration on the target systems. Unauthorized scanning may violate laws and ethical guidelines.
