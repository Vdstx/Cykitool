WPScan

About WPScan

WPScan is a black box WordPress vulnerability scanner written in Ruby. It is designed to help security professionals and WordPress administrators identify potential security issues within WordPress installations, including vulnerable plugins, themes, and weak passwords. WPScan uses a database of known vulnerabilities to check against the target site.


Installation

WPScan requires Ruby and RubyGems to be installed on your system.


Installing WPScan on Debian-based Systems:

sudo apt install ruby ruby-dev
sudo gem install wpscan

Updating WPScan:

sudo gem update wpscan

API Token

WPScan uses an API to fetch the latest vulnerability data. You need to register for a free API token at wpscan.com/register and include it in your scans using the --api-token option or by setting it in the configuration file.


Examples of WPScan Usage

Basic Scan of a WordPress Site:

wpscan --url https://example.com --api-token YOUR_API_TOKEN
  • --url: Specifies the target URL.
  • --api-token: Your WPScan API token.

Enumerating Plugins:

wpscan --url https://example.com --enumerate p --api-token YOUR_API_TOKEN
  • --enumerate p: Enumerates installed plugins.

Enumerating Users:

wpscan --url https://example.com --enumerate u --api-token YOUR_API_TOKEN
  • --enumerate u: Enumerates WordPress usernames.

Password Brute-Forcing:

wpscan --url https://example.com --passwords /path/to/wordlist.txt --api-token YOUR_API_TOKEN
  • --passwords: Specifies the path to a password wordlist for brute-forcing.

Useful Options

Option Description
--url The target URL to scan.
--update Update WPScan's database to the latest version.
--enumerate Enumeration options (e.g., p for plugins, t for themes, u for users).
--plugins-detection Specify the method for plugin detection (mixed, passive, aggressive).
--api-token Your WPScan API token for vulnerability data.
--random-user-agent Use a random User-Agent for each scan.
--user-agent Set a custom User-Agent string.
--proxy Use a proxy for requests (e.g., http://127.0.0.1:8080).
--cookies Set cookies to be used in requests.
--headers Set custom HTTP headers (e.g., 'Accept: */*').
--passwords Path to password wordlist for brute-forcing.
--usernames Specify usernames for password brute-forcing.
--threads Number of threads to use (Default: 5).
--output Save scan results to a file.
--format Format of the output file (cli, json, cli-no-colour).

Troubleshooting and Tips

  • API Rate Limits: Free API tokens have rate limits. Plan scans accordingly or consider upgrading for higher limits.
  • Customizing Scans: Use enumeration options to focus on specific areas (e.g., --enumerate ap,at for all plugins and themes).
  • Respect Target Resources: Adjust the number of threads with --threads to avoid overloading the target server.
  • Proxy Usage: Use the --proxy option to route traffic through a proxy like Burp Suite for analysis.
  • User Enumeration: Be aware that user enumeration may not always be possible if the site has protections in place.
  • Updates: Regularly update WPScan and its vulnerability database with --update to ensure the latest data is used.

Legal Considerations

Ensure you have explicit permission to scan and test the target WordPress installations. Unauthorized scanning may violate laws, regulations, and ethical guidelines.
