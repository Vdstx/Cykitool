Wireshark

About Wireshark

Wireshark is one of the most widely used network protocol analyzers in the world. It is an open-source tool that captures network traffic in real-time, allowing security professionals, network administrators, and ethical hackers to inspect data packets in detail. Wireshark supports hundreds of network protocols and is essential for troubleshooting, monitoring, and understanding what is happening inside a network.

Wireshark can capture traffic on different interfaces (Ethernet, Wi-Fi, etc.) and helps analyze issues such as latency, packet loss, security threats, and misconfigurations. It also decodes encrypted traffic in certain conditions, providing a comprehensive view of network activity.


Installation

Wireshark is available for multiple platforms (Windows, macOS, Linux). Installation is simple, and it requires appropriate permissions (administrator/root) to capture network traffic.

Linux installation (Ubuntu/Debian):

sudo apt update && sudo apt install wireshark

Windows installation:

  1. Download the installer from the official site: Wireshark Downloads.
  2. Run the installer and select the necessary options, including installation of WinPcap (for packet capturing).
  3. Launch Wireshark and select a network interface to start capturing traffic.

Examples of Wireshark Usage

Capturing Network Traffic

To start capturing network traffic:

  1. Open Wireshark and choose the network interface (e.g., Ethernet, Wi-Fi).
  2. Click on the green "Start Capturing" button.
  3. Wireshark will now display all the network traffic passing through the chosen interface.

While capturing, Wireshark organizes the traffic into a table, with each row representing a single packet. It provides details such as the packet number, time, source, destination, protocol, length, and information about the packet.


Filtering Packets

Wireshark can capture thousands of packets in seconds, but using filters helps narrow down relevant data. Here are some useful filters:

  • ip.addr == 192.168.1.1: Capture packets sent from or to a specific IP address.
  • tcp.port == 80: Capture HTTP traffic (which uses port 80 by default).
  • dns: Filter only DNS-related traffic.
  • http.request: Show only HTTP requests, excluding responses.
  • tls: Capture TLS-encrypted traffic (formerly known as SSL).

Packet Analysis Example

Once a packet is selected, the details are displayed in three sections:

  • Packet List: A summary of all captured packets in chronological order.
  • Packet Details: The contents of the selected packet, broken down by protocol layers (Ethernet, IP, TCP/UDP, etc.).
  • Packet Bytes: A raw hexadecimal view of the packet’s data.

For example, in an HTTP packet, you can inspect headers such as the request method (GET, POST), URL, and status codes (200 OK, 404 Not Found). In a TCP packet, you can see details about the connection, like the sequence number and flags (SYN, ACK, FIN).


Analyzing Encrypted Traffic

Wireshark can decrypt SSL/TLS traffic under certain conditions if you have access to the server’s private key or use session keys exported from browsers. To decrypt TLS traffic:

  1. Go to Edit > Preferences > Protocols > TLS.
  2. Specify the location of the server’s private key (for HTTPS decryption) or the session key log file (for browser session keys).
  3. Wireshark will automatically decrypt the traffic and display the decoded packets.

Use Cases of Wireshark

  • Network Troubleshooting: Identify issues such as packet loss, high latency, and network congestion.
  • Security Analysis: Detect malicious activities, man-in-the-middle attacks, or traffic anomalies.
  • Protocol Analysis: Understand how various protocols (HTTP, FTP, DNS, DHCP, etc.) function in real-time network traffic.
  • Performance Monitoring: Analyze bandwidth usage, data flow between devices, and optimize network performance.
  • Forensics: In the event of a security breach, Wireshark captures can help reconstruct and analyze what happened on the network.

Advanced Wireshark Features

  • Coloring Rules: Wireshark allows you to assign colors to different packet types (e.g., HTTP, DNS, ARP) to make the analysis easier.
  • Follow TCP Stream: Rebuilds a conversation from the individual TCP packets, allowing you to see the complete communication (e.g., HTTP requests and responses).
  • IO Graphs: Graph network traffic over time to visually identify spikes in usage or packet drops.
  • Packet Reassembly: Wireshark can reassemble fragmented packets to display complete data streams, especially useful for protocols like HTTP and FTP.
  • Exporting Packets: Export packet captures in various formats, including .pcap and .csv, for further analysis or sharing.
