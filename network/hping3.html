hping3

About hping3

hping3 is an open-source packet generator and analyzer for the TCP/IP protocol. It is used for network testing, firewall testing, advanced port scanning, and many other tasks. hping3 allows users to craft custom TCP/IP packets and analyze network responses, making it an essential tool for network administrators and security professionals.

Unlike the traditional ping command, which uses ICMP echo requests, hping3 supports TCP, UDP, ICMP, and RAW-IP protocols, giving users more flexibility in how they interact with network devices.


Installation

hping3 is available for Unix-like operating systems, including Linux and macOS. Below are the installation steps for common platforms.

Linux Installation (Ubuntu/Debian):

sudo apt update && sudo apt install hping3

macOS Installation:

brew install hping

Note: This requires Homebrew to be installed on your macOS.


Basic Usage of hping3

hping3 is a command-line tool that typically requires root or sudo privileges to send and receive custom network packets.

Performing a Basic Ping with TCP

To send TCP SYN packets to port 80 of a target host:

sudo hping3 -S -p 80 <target_ip>

This sends TCP SYN packets to port 80 and waits for replies, similar to how ping works with ICMP.


Sending ICMP Echo Requests

To perform a traditional ICMP ping:

sudo hping3 <target_ip> --icmp

UDP Scan on a Specific Port

To send UDP packets to a specific port:

sudo hping3 <target_ip> --udp -p 53

This sends UDP packets to port 53, which is commonly used for DNS services.


Setting Packet Rate and Count

To send a specific number of packets at a defined rate:

sudo hping3 <target_ip> -i u1000 -c 10

This sends 10 packets (-c 10) at a rate of one packet every 1000 microseconds (-i u1000), i.e., 1000 packets per second.


Advanced hping3 Techniques

Firewall Testing

hping3 can be used to test firewall rules by sending packets with specific flags and observing the responses.

For example, to send TCP ACK packets to port 80:

sudo hping3 -A -p 80 <target_ip>

Traceroute with hping3

To perform a traceroute using TCP packets:

sudo hping3 --traceroute -V -S -p 80 <target_ip>

This traces the route to the target using TCP SYN packets to port 80.


Simulating a Specific Operating System

To set the TTL (Time To Live) and window size to match a specific OS:

sudo hping3 <target_ip> -S -p 80 --ttl 128 --win 65535

This can be used to simulate packets from a Windows machine.


Sending Files Over Network

hping3 can send files by fragmenting them into packets:

sudo hping3 <target_ip> -E file.txt --sign signature

Note: This is an advanced usage and requires a corresponding hping3 listener to reassemble the data.


Use Cases of hping3

  • Network Testing: Measure network performance, latency, and packet loss.
  • Firewall Auditing: Test firewall rules and intrusion detection systems (IDS) by sending crafted packets.
  • Advanced Traceroute: Perform traceroute operations using TCP, UDP, or ICMP with more control over packet parameters.
  • Protocol Analysis: Analyze how different network protocols respond under various conditions.
  • Security Assessments: Identify potential vulnerabilities in network configurations.

Best Practices and Tips

  • Use Responsibly: Only use hping3 on networks and systems for which you have explicit permission to test.
  • Understand Legal Implications: Unauthorized network scanning or packet crafting can be illegal and unethical.
  • Start with Basic Commands: If you're new to hping3, begin with simple commands to familiarize yourself with its functionality.
  • Combine with Other Tools: Use hping3 alongside tools like Wireshark for comprehensive network analysis.
  • Refer to Official Documentation: hping3 has extensive documentation that can help you understand advanced features.

Common hping3 Options Summary

Option Description
-S Set SYN flag
-A Set ACK flag
--udp Use UDP mode
--icmp Use ICMP mode
-p <port> Set destination port
-i <interval> Packet send interval (e.g., -i u1000 for 1000 microseconds)
-c <count> Stop after sending (and receiving) a certain number of packets
--traceroute Enable traceroute mode
-E <file> Send data from a file
--sign <text> Add a signature to outgoing packets
--ttl <value> Set Time To Live value
--win <size> Set TCP window size

Legal and Ethical Considerations

hping3 is a powerful tool that can generate custom network traffic. It is crucial to use hping3 responsibly and ethically. Unauthorized use of hping3 to scan networks or send crafted packets can lead to legal consequences and disrupt network services. Always ensure you have explicit permission to perform network tests and comply with all applicable laws and organizational policies.
