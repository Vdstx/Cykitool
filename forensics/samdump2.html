samdump2

About samdump2

samdump2 is a forensic tool used to extract password hashes from Windows Security Account Manager (SAM) databases. It works by reading the SYSTEM and SAM registry hive files, allowing investigators to obtain NTLM and LM password hashes for analysis. This tool is commonly used in digital forensics and security auditing to assess password strength and recover lost passwords.


Installation

On Debian-based Linux distributions:

sudo apt-get install samdump2

To build from source:

git clone https://github.com/moyix/samdump2
cd samdump2
make
sudo make install

Examples of samdump2 Usage

Extracting Password Hashes from SAM and SYSTEM Files:

samdump2 SYSTEM SAM > hashes.txt

  • SYSTEM: The SYSTEM registry hive file containing the SYSKEY.
  • SAM: The SAM registry hive file containing user account information.
  • hashes.txt: The output file where the extracted hashes will be saved.

Using samdump2 with a Different Output Format:

samdump2 SYSTEM SAM | cut -d ":" -f1,4 > ntlm_hashes.txt

  • cut -d ":" -f1,4: Extracts the username and NTLM hash fields.
  • ntlm_hashes.txt: The file containing formatted NTLM hashes.

Useful Options

  • -o: Specifies the output file for the hashes.
  • -d: Enables debug mode for detailed output.
  • -h: Displays the help message with usage instructions.

Troubleshooting and Tips

  • Access Permissions: Ensure you have the necessary permissions to read the SAM and SYSTEM files, which may require administrative privileges.
  • Acquiring Registry Hives: Use tools like reg save or boot into a live environment to copy the SAM and SYSTEM files from a Windows machine.
  • Hash Cracking: After extracting the hashes, use password cracking tools like John the Ripper or Hashcat to attempt to recover plaintext passwords.
  • Compatibility: samdump2 is compatible with Windows NT, 2000, XP, Vista, 7, and some versions of Windows 8 and 10.

Legal Considerations

Always ensure you have proper authorization before accessing and extracting password hashes from systems. Unauthorized access to computer systems and data is illegal and unethical.
