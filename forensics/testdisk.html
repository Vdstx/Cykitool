TestDisk

About TestDisk

TestDisk is a powerful open-source data recovery utility designed to help recover lost partitions and make non-booting disks bootable again. It works with a variety of filesystems and operating systems, making it a versatile tool for both professionals and home users. TestDisk can fix partition tables, recover deleted partitions, rebuild boot sectors, and more.


Installation

On Debian-based Linux distributions:

sudo apt-get install testdisk

Examples of TestDisk Usage

Recovering a Lost Partition:

  1. Run TestDisk as root or with sudo:
    2. sudo testdisk
  2. Select Create to create a new log file.
  3. Choose the disk that contains the lost partition.
  4. Select the partition table type (usually Intel for PC).
  5. Select Analyse to scan for partitions.
  6. Proceed with Quick Search. If the partition is not found, use Deeper Search.
  7. Once the lost partition is found, select it and press p to list files (optional).
  8. Press Enter to continue, then Write to save the partition table.
  9. Confirm and reboot the system.

Rebuilding a Corrupted NTFS Boot Sector:

  1. Run TestDisk:
    2. sudo testdisk
  2. Select Create or Append to create or add to a log file.
  3. Select the disk and partition with the corrupted boot sector.
  4. Choose Advanced and select the partition.
  5. Select Boot, then choose Rebuild BS to rebuild the boot sector.
  6. Confirm and write the new boot sector.

Copying Files from a Damaged Partition:

  1. Navigate to the desired partition in TestDisk.
  2. Press p to list files and directories.
  3. Select files/directories using : and press c to copy.
  4. Choose the destination directory to save the copied files.

Useful Options

  • Analyse: Scans for lost partitions and recovers them.
  • Advanced: Access advanced filesystem utilities like boot sector recovery.
  • Geometry: Modify disk geometry settings if detected incorrectly.
  • Options: Set various options like enabling expert mode.
  • MBR Code: Rewrites the Master Boot Record code.
  • EFI GPT: Support for GUID Partition Table (GPT) disks.

Troubleshooting and Tips

  • Read-Only Mode: TestDisk operates in read-only mode until changes are explicitly written, minimizing the risk of data loss.
  • Backup Data: If possible, backup the entire disk or partition before making changes.
  • Documentation: Refer to the official documentation for detailed instructions on specific recovery scenarios.
  • Disk Identification: Ensure you select the correct disk to avoid modifying the wrong device.
  • File System Support: TestDisk supports various filesystems including FAT12/16/32, NTFS, exFAT, ext2/3/4, and HFS+.

Example: Recovering a Deleted Partition on a USB Drive

If you've accidentally deleted a partition on a USB drive:

  1. Insert the USB drive and identify its device name (e.g., /dev/sdc).
  2. Run TestDisk:
    3. sudo testdisk
  3. Select Create to start a new log.
  4. Choose the USB drive from the list.
  5. Select the appropriate partition table type.
  6. Proceed with Analyse and Quick Search.
  7. When the deleted partition appears, select it and press p to verify files.
  8. If correct, proceed to write the partition table and reboot if necessary.

Legal Considerations

Ensure you have the legal right and authorization to recover data from the device. Unauthorized recovery of data may violate privacy laws and regulations.
