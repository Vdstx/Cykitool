Binwalk

About Binwalk

Binwalk is a fast and easy-to-use tool for analyzing, reverse engineering, and extracting firmware images. It scans binary files for embedded files and executable code, leveraging signatures, magic numbers, and other heuristics to identify and extract them. Binwalk is commonly used in the field of embedded systems security, digital forensics, and reverse engineering.


Installation

On Debian-based Linux distributions:

sudo apt-get install binwalk

To install the latest version from source:

git clone https://github.com/ReFirmLabs/binwalk.git
cd binwalk
sudo python3 setup.py install

Examples of Binwalk Usage

Scanning a Firmware Image:

binwalk firmware.bin
  • Performs a signature scan on firmware.bin to identify embedded files and code.

Extracting Embedded Files Automatically:

binwalk -e firmware.bin
  • -e or --extract: Automatically extracts identified files.

Recursive Extraction:

binwalk -Me firmware.bin
  • -M or --matryoshka: Enables recursive extraction (extract files within extracted files).
  • -e: Extracts files as before.

Viewing Entropy Analysis:

binwalk --entropy firmware.bin
  • --entropy: Generates an entropy graph to help identify compressed or encrypted data.

Carving Files Based on Signatures:

binwalk --dd='png image:png' imagefile.img
  • --dd: Defines custom extraction rules.
  • 'png image:png': Extracts all PNG images and saves them with a .png extension.

Useful Options

  • -B or --signature: Performs a binary signature scan (default behavior).
  • -R or --raw: Scans for common raw compressed data.
  • -A or --opcodes: Scans for executable code signatures.
  • -E or --entropy: Performs entropy analysis.
  • -D or --dd: Extracts data matching a specified signature.
  • -X or --hexdump: Displays a hexdump of the specified file.
  • -I or --include: Includes a custom signature file for scanning.
  • -r or --raw-bytes: Extracts raw bytes from the target file.

Troubleshooting and Tips

  • Permissions: Ensure you have read permissions on the files you are analyzing.
  • Dependencies: Install additional extractors to improve extraction capabilities (e.g., sudo apt-get install p7zip-full).
  • Custom Signatures: Create custom signature files for proprietary formats using the --include option.
  • Analyzing Entropy: Use entropy analysis to spot encrypted or compressed sections within firmware.
  • Verbose Output: Use -v for verbose output to get more detailed information during scanning.

Example: Extracting Files from a Firmware Image

Suppose you have a firmware image named router_firmware.bin and you want to extract all possible files:

binwalk -e router_firmware.bin

This command will create a directory named _router_firmware.bin.extracted containing all the extracted files.


To perform a recursive extraction, including archives within the firmware:

binwalk -Me router_firmware.bin

Legal Considerations

Ensure you have the legal right and authorization to analyze and extract firmware images. Unauthorized reverse engineering or extraction may violate licensing agreements and laws.
