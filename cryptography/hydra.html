Hydra

About Hydra

Hydra (also known as THC-Hydra) is a popular, open-source network login cracker that supports a wide variety of protocols, making it an essential tool for penetration testers and cybersecurity professionals. It is mainly used to perform brute-force and dictionary-based attacks against various authentication services like SSH, FTP, HTTP, and more. Hydra is highly efficient due to its ability to test multiple password combinations at once (parallel attacks). Installation

Linux installation

sudo apt install hydra

Alternatively, still on linux

git clone https://github.com/vanhauser-thc/thc-hydra
cd thc-hydra
./configure
make
sudo make install

Examples of Hydra Usage

Brute-force attack on an SSH service:

hydra -L usernames.txt -P passwords.txt ssh://192.168.1.100

  • -L usernames.txt: Path to the file containing a list of usernames.
  • -P passwords.txt: Path to the file containing a list of passwords.
  • ssh://192.168.1.100: Target IP address for the SSH service.

Brute-force attack on an FTP service:

hydra -l admin -P passwords.txt ftp://192.168.1.100

  • -l admin: Single username (admin in this case).
  • ftp://192.168.1.100: FTP service URL (default port 21).

HTTP login page brute force:

hydra -l admin -P passwords.txt http-post-form "/login.php:username=^USER^&password=^PASS^:F=incorrect"

  • /login.php: Path to the login form.
  • ^USER^ and ^PASS^: Placeholders for username and password in the POST request.
  • F=incorrect: Fail condition (the text Hydra looks for when login fails).

Brute-force attack on an RDP service:

hydra -t 4 -V -f -l user1 -P passlist.txt rdp://192.168.1.100

  • rdp://192.168.1.100: RDP service on the target host.

Useful options

  • -l: Specify username list or a single username, respectively.
  • -p: Specify password list or a single password, respectively.
  • -s: Specify the target port (if different from the default)
  • -f: Stop after the first valid login.
  • -V: Verbose mode, shows details of each attempt.
  • -t: Number of parallel tasks/threads.
  • -e nsr: Try "no password," "same as username," and "reverse username" as passwords.
  • -M: Use a file containing multiple target IP addresses.
  • -o: Output results to a file.
