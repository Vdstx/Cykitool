RsaCtfTool

About RsaCtfTool

RsaCtfTool is a Python-based tool designed to help in the exploitation and solving of RSA-based cryptographic challenges. It is widely used in Capture The Flag (CTF) competitions and for educational purposes in cryptanalysis. The tool is capable of factoring large integers, finding private keys, and decrypting ciphertexts using common vulnerabilities found in RSA implementations, such as small public exponents or faulty key generation processes.


Installation

Linux installation using pip:

pip install git+https://github.com/Ganapati/RsaCtfTool

Alternatively, you can clone the repository and install dependencies:

git clone https://github.com/Ganapati/RsaCtfTool
cd RsaCtfTool
pip install -r requirements.txt

Examples of RsaCtfTool Usage

Decrypt ciphertext with private key:

python3 RsaCtfTool.py --private --key private_key.pem --uncipherfile ciphertext.enc

  • --private: Specifies the use of a private key for decryption.
  • --key: Path to the private key file.
  • --uncipherfile: The encrypted file to decrypt.

Cracking RSA when given n and e (public key parameters):

python3 RsaCtfTool.py --publickey public_key.pem --attack all

  • --publickey: Path to the public key file.
  • --attack: Runs multiple attacks (like factorization, Wiener's attack, etc.) to try and retrieve the private key.

Using RsaCtfTool for small exponent attack (common in CTFs with small e values):

python3 RsaCtfTool.py --publickey public_key.pem --attack cube_root

  • cube_root: Specific attack for small e values (e.g., 3).

Factoring a large modulus n to retrieve private keys:

python3 RsaCtfTool.py --publickey public_key.pem --attack factordb

  • factordb: Leverages the Factordb.com database to find the factors of the modulus.

Common RSA Vulnerabilities

  • Small public exponent: RSA implementations using small public exponents (e.g., e=3) can be vulnerable to attacks like cube-root attacks if the ciphertext is small.
  • Common modulus: When two different public keys share the same modulus n but different exponents, it is possible to recover the private key using the Extended GCD algorithm.
  • Wiener's attack: When the private exponent d is too small, Wiener's attack can recover the private key.
  • Factorable modulus: If n (modulus) is a product of small prime numbers, factoring algorithms can retrieve the primes and, consequently, the private key.

Useful Options

  • --publickey: Specify the public key file to be used in attacks.
  • --private: Specify the private key file for decryption.
  • --attack: Run specific or multiple attacks to recover private keys or decrypt ciphertexts.
  • --uncipher: Specify the file to decrypt using the provided private key.
  • --createpub: Create a public key from modulus and exponent values.
  • --dumpkey: Dumps information about a public or private key.
  • -v: Enable verbose mode for detailed output.
