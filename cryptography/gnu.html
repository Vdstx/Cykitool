GnuPG

About GnuPG

GnuPG (Gnu Privacy Guard) is a free and open-source encryption tool that implements the OpenPGP standard. It is widely used for encrypting and signing data and communications. GnuPG is a vital tool for anyone concerned with privacy, allowing for secure email exchanges and file encryption. It uses a combination of symmetric-key and public-key cryptography to safeguard information, and it supports a variety of encryption algorithms such as RSA and AES. GnuPG is favored for its reliability and is a critical component in many security infrastructures.


Installation

Linux installation

sudo apt install gnupg

Alternatively, you can compile from source:

git clone https://gnupg.org/download/
cd gnupg
./configure
make
sudo make install

Examples of GnuPG Usage

Generating a new key pair:

gpg --full-generate-key

  • This will prompt you to choose encryption options such as key type, key size, and expiration date.

Encrypting a file:

gpg --output encrypted_file.gpg --encrypt --recipient recipient@example.com file.txt

  • --output: Specifies the name of the encrypted file.
  • --encrypt: Command to encrypt the file.
  • --recipient: Specifies the recipient's public key.

Decrypting a file:

gpg --output decrypted_file.txt --decrypt encrypted_file.gpg

  • --output: Specifies the name of the decrypted file.
  • --decrypt: Command to decrypt the file using your private key.

Signing a file:

gpg --output signed_file.gpg --sign file.txt

  • --sign: Creates a digital signature for the file.

Useful Options

  • --armor: Outputs ASCII armored encrypted files for easy readability.
  • --list-keys: Lists all public keys in your keyring.
  • --list-secret-keys: Lists all private keys in your keyring.
  • --import: Imports a public or private key from a file.
  • --export: Exports a public key for sharing.
  • --trust-model: Specifies the trust model for key verification.
  • --symmetric: Encrypts a file using a symmetric cipher (password-based encryption).
